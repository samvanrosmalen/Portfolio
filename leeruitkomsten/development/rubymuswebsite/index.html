Leeruitkomsten Development Website Ruby Mus

Website Ruby Mus

Na het maken van een design voor de website van Ruby Mus zijn we in sprint 2 met onze groep begonnen aan het developpen van de website, hiervoor hadden we een duidelijke taakverdeling gemaakt. Mijn foucs lag bij het maken van de header en het animeren hiervan.

Header

Voor het maken van de header heb ik gebruik gemaakt van twee foto's, een foto van Ruby Mus die we tijdens de fotoshoot hebben gemaakt en die ik daarna heb bewerkt in Adobe Photoshop en een foto van haar namen. Deze heb ik gemaakt in Adobe Illustrator. Voor sprintoplevering 2 hebben we eerst een webiste gemaakt zonder animaties, dit hebben we gedaan zodat we eerst feedback konden krjgen op de website voordat we met animaties gingen werken. In spirnt 3 zijn we begonnen met het animeren van de website.

Animaties

Voor het animeren van de header heb ik gebruik gemaakt van de Animate.css library. Dit is een makkelijke library om animaties toe te voegen. Door de afbeelding van Ruby Mus haar namen op te splitsen in allemaal losse afbeeldingen en hier verschillende animatie tijden aan toe te voegen heb ik dit effect gemaakt. Meer hierover kun je hier op de interactive media projectpagina lezen.

Wat heb ik geleerd?

Ik heb voor dit project gewerkt met animatie tijden, hiermee had ik nog niet gewerkt. Ik wilde eerst werken met delays vanuit Animate.css alleen kwam ik er na veel proberen en research doen achter dat hier een limiet op zit. Ik heb daarom voor het alternatief van animatie tijd gekozen, dit geeft hetzelfde effect als ik in mijn hoofd had.

Wat kon beter?

Het positioneren van de foto's ging niet helemaal goed, dit komt doordat ik een andere resolutie scherm heb omdat ik op een Macbook werk. Daarom zag het er bij mijn groepsgenoten heel anders uit dan bij mij, na wat gepuzzel zag het er bij iedereen hetzelfde uit.
